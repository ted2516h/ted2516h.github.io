在Lua编程语言中，==%==、==//== 和 ==*== 是三个具有特定功能的运算符，但它们各自的功能和存在性需要一些澄清，因为Lua并不直接支持某些在其他编程语言中常见的运算符，如//（整数除法运算符，至少在Lua的标准版本中不是）。下面是这些运算符在Lua中的功能说明：

  1. % 运算符：
    • % 是取模运算符，也称为余数运算符。它用于计算两个数相除后的余数。
    • 例如：10 % 3 的结果是 1，因为10除以3商3余1。
  2. // 运算符：
    • 在Lua的标准版本中，// 不是一个有效的运算符。Lua不支持直接的整数除法运算符。
    • 如果你想要进行整数除法（即丢弃小数部分），你需要使用数学库中的函数，如 math.floor(a / b) 或 math.ceil(a / b)（取决于你想要的舍入行为），或者直接使用除法运算符 / 并处理结果（Lua中的除法总是返回浮点数）。
    • 注意：在某些Lua的扩展或特定版本中，可能定义了//运算符，但这不是Lua语言标准的一部分。
  3. * 运算符：
    • * 是乘法运算符。它用于计算两个数的乘积。
    • 例如：5 * 3 的结果是 15。

因此，在Lua中，你会经常使用 % 和 * 运算符进行模运算和乘法运算，但如果你需要整数除法，你需要采用其他方法，如使用 math.floor 函数结合普通的除法运算符 /。

从数学上看，这两个表达式都会随着t的变化在1和3之间变化。但是，它们变化的频率是不同的。t%60//30*2+1==每60个单位时间变化一次==（因为取模60），而t%30//15*2+1每==30个单位时间就变化一次==（因为取模30）。

  • 如果这两个表达式被用作动画的时间或进度参数，并且动画的播放速度与这些参数的变化频率相关，那么这两个表达式会导致不同的动画播放速度。具体来说，t%30//15*2+1会导致动画的播放速度是t%60//30*2+1的两倍，因为前者每30个单位时间变化一次，而后者每60个单位时间变化一次。

t%60//20* ==4==+1 1，5，9 ==4==倍

PS:

  1. **TIC()**：这是游戏的主循环函数，每帧都会调用一次。你可以在这里处理游戏逻辑和渲染。
  2. **cls(color)**：清除屏幕并用指定颜色填充。
  3. **print(text, x, y, color)**：在指定位置绘制文本。
  4. **spr(id, x, y, colorkey, scale, flip, rotate)**：绘制精灵。
  5. **map(cell_x, cell_y, sx, sy, w, h, colorkey)**：绘制地图的一部分。
  6. **sfx(id, note, duration, channel, volume, speed)**：播放声音效果。
  7. **music(track, frame, row, loop)**：播放音乐。
